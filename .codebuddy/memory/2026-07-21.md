# 2026-07-21

## 铺路逻辑新增「夜间 ×3」
- 需求（更正）：白天/浅夜（08:00–次日 01:00）接力 3分/格、自动 3分10秒/格；深夜（01:00–08:00）翻 3 倍：接力 9分/格、自动 9分10秒/格。
- 关键：×3 只乘「3 分钟」基础部分，自动铺路固定 +10 秒「行路」不随昼夜翻倍（夜间 auto = 3×3分 + 10秒 = 9分10秒）。
- 核心函数集中在 `utils/taskTime.ts`：`PAVING_CELL_MIN=3`、`PAVING_OVERHEAD_SEC`（auto=10/relay=0）、`PAVING_NIGHT_MULT=3`、`PAVING_NIGHT_START=60`、`PAVING_NIGHT_END=480`、`paceSecAt(mode,absMin)`（夜间仅基础分钟 ×3）、`pavedEndAbs`、`pavedCountBetween`。
- 移除旧的固定常量 `PAVING_PACE_MIN`（taskTime）与本地 `PAVING_PACE`（ganttViewBuild、ganttCanvasDraw），三处统一复用 taskTime。
- `useTaskEditor.ts`：模板填充 / `onCountValueInput` / `syncPaveFromEndTime` / `paveDurationText` 均改用 pavedEndAbs / pavedCountBetween（昼夜 pace）。
- `ganttViewBuild.ts` 的 `computePavedCount` / `computeMergedPaveTicks`，`ganttCanvasDraw.ts` 的 `computeTicks`：改为逐格按昼夜 pace 累计（双指针取 interval 网格上的累计格数），O(格数+刻度数)。
- 注意：大营（攻城/同盟）的夜间行为是「暂停」（`campEndTime`，00:00–09:00），与铺路 ×3 是两套独立机制，不要混淆。
- `docs/事务预设.md` 第六节/七节已同步（9分/格 + 夜间×3 已实现）。`taskTemplates.ts` 的 pavingOptions 文案更新。
