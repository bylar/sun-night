# 事务预设（模板）说明

> 本文档系统梳理 `sun-night` 当前「事务预设（模板）」的全部类型、填充逻辑、表单字段与规则。
> 内容以**实际代码实现**为准（来源：`composables/useTaskEditor.ts`、`components/TaskEditDialog.vue`、`components/TemplatePicker.vue`、`types/gantt.ts`、`utils/ganttLayout.ts`）。
> 与 `README.md` / `types/gantt.ts` 注释中不一致的铺路节奏，见文末「实现差异说明」。

---

## 一、概念与入口

- 新增事务时，先弹出 **模板选择弹层**（`TemplatePicker.vue`，九宫格），点击任一模板即自动填充默认参数，再进入 **事务编辑弹窗**（`TaskEditDialog.vue`）做二次编辑。
- 模板数据来自 `useTaskEditor.ts` 中的 `TEMPLATES` 常量（类型为 `TaskTemplate`）。
- 流程：`GanttChart` 底部加号 → `editor.openAdd()` 打开模板弹层 → 选中 `pick(t)` → `editor.pickTemplate(t)` 填充 `editForm` 并打开编辑弹窗。

**当前预设共 6 种**：攻城大营、同盟大营、宣战、自动铺路、接力铺路、自定义事务。
> 注：「铺路（pave）」手动模板已移除——铺路仅保留「自动铺路 / 接力铺路」两种（见第五节）。

## 二、通用表单字段（editForm 全量）

编辑弹窗共享同一个 `editForm`（由 `useTaskEditor` 提供，`provide('taskEditor')`）。所有字段如下：

| 字段 | 类型 | 含义 |
|------|------|------|
| `name` | `string` | 事务名称（必填，保存前校验非空） |
| `startTime` | `string` | 开始时间，格式 `"YYYY-MM-DD HH:mm"` |
| `endTime` | `string` | 结束时间（可选；为空 = 瞬时任务） |
| `priority` | `'high'\|'medium'\|'low'` | 优先级，默认 `medium` |
| `description` | `string` | 备注（可选） |
| `isHighlighted` | `boolean` | 高亮重点（置顶强调） |
| `color` | `string` | 卡片颜色（模板带默认值，可改） |
| `icon` | `string` | Vant 图标名（模板带默认值） |
| `pavingMode` | `''\|'auto'\|'relay'` | 铺路模式（空=不铺路） |
| `countMode` | `'count'\|'time'` | 铺路填写方式：按格数 / 按时长 |
| `countValue` | `number` | 铺路数值（格数或分钟，依 `countMode`） |
| `count` | `string\|number` | 非铺路时的手动计数（右侧数字） |
| `durationMin` | `number` | 模板默认/固定时长（分钟），用于按下开始时间推算结束时间 |
| `minDurationMin` | `number` | 最短时长（分钟），手动改结束时间时约束下限（当前未被任何模板使用） |
| `stepMin` | `number` | 结束时间粒度（分钟），手动改结束时间时吸附到该步长（当前未被任何模板使用） |
| `customEnd` | `boolean` | **是否允许用户改结束时间**：`false`=结束时间由系统按规则推算并仅在表单展示（用作大营标志）；`true`=用户可自由编辑 |
| `template` | `TaskTemplateId` | 来源模板 id（仅作标记，不参与渲染） |

**颜色预设（`COLOR_PRESETS`，10 色）**：`#07c160` `#ee0a24` `#ff976a` `#1989fa` `#7232dd` `#ffcd00` `#323233` `#969799` `#00b8d4` `#e91e63`；并在此之外支持原生取色器自定义。

**优先级显示**：高(danger)/中(warning)/低(default)，按钮点击在 高→中→低 间循环切换。

## 三、通用规则（适用所有类型）

### 1. 保存校验
- `name` 为空 → `showToast('请输入事务名称')` 并中断保存。
- 只读身份（`canEdit=false`，分享链接只读）：保存/删除按钮禁用，主入口已拦截编辑点击（防御性校验，服务端仍二次校验）。

### 2. 保存时的字段取舍（`saveTask`）
- `pavingMode`：空串存为 `null`。
- `countMode` / `countValue`：仅当 `pavingMode` 非空时才落库，否则 `undefined`。
- `durationMin`：仅当 `> 0` 落库；`minDurationMin` 仅当 `> 0`；`stepMin` 仅当 `> 1`（当前模板均不设置，故不落库）。
- `customEnd`：始终落库。
- `count`：
  - 铺路模式：取 `count`（数字）否则 `countValue`，再否则 `undefined`；
  - 非铺路模式：`count === ''` → `undefined`，否则 `Number(count)`。

### 3. 时间选择器
- 开始/结束时间均为 `日期 + 时 + 分` 三级 Picker，1 分钟精度。
- 日期范围：今天前 60 天 ~ 后 180 天（共 241 天）。
- 结束时间可跨天；清除结束时间 = 设为瞬时任务。
- 默认开始时间：今天日期 + 就近 5 分钟对齐的当前时刻。

### 4. 按下开始时间的联动（`onStartConfirm`）
- 若 `durationMin > 0`：
  - `customEnd === false`（大营）→ 结束时间按 **夜间暂停** 规则推算（`campEndTime`）；
  - 否则 → `endTime = startTime + durationMin`（固定时长）。
- 否则（铺路模式）：调用 `syncPaveFromEndTime()` 由结束时间反推铺路格数。

### 5. 结束时间可编辑性
- `customEnd === true`（宣战、铺路、自定义等）：结束时间可点击打开 Picker 编辑，可清除为瞬时。
- `customEnd === false`（攻城大营、同盟大营）：结束时间**仅在表单展示**，不可点击、不可清除——由开始时间与固定时长按夜间暂停推算。

### 6. 手动改结束时间的约束（`onEndConfirm`，仅 `customEnd===true` 时可用）
- 若设置了 `minDurationMin`（`> 0`）：`结束 - 开始 < min` 时强制拉到 `start + min`。
- 若设置了 `stepMin`（`> 1`）：结束时间吸附到 `start + round((m-s)/step)*step`。
- 若 `m < s`：取 `s`（即瞬时）。

### 7. 铺路「按格数 / 按时长」互转
- 仅当 `pavingMode` 非空时显示「填写方式」切换（按格数 / 按时间）。
- `countMode = 'count'`（按格数）：显示「格数」输入框 → `onCountValueInput` 实时 `endTime = startTime + round(countValue * PACE[pavingMode])`，并 `count = countValue`。
- `countMode = 'time'`（按时间）：显示「铺路时长」文案（`paveDurationText`，形如 `X时Y分 · 约N格`），点按打开结束时间选择器。
- `syncPaveFromEndTime()`：`pavingMode` 非空且 `countMode !== 'time'` 时，由 `endTime - startTime` 反推 `count = round(dur / PACE[pavingMode])`。
- 选铺路模式（`onPavingConfirm`）：若由空改为铺路，自动补 `countMode='count'`、`countValue=10`，并即时重算格数/结束时间。

### 8. 权限
- 编辑类交互受 `inject('canEdit')` 控制；只读（viewer）态保存/删除禁用。服务端对所有写操作二次校验 `canEdit` / `isOwner` / 管理员身份。

## 四、大营「夜间暂停」推算规则

攻城大营、同盟大营等「大营」类事务在夜间不推进，`customEnd=false`：
- **仅定义开始时间**；结束时间由系统按固定时长推算，**仅在表单展示**不可编辑。
- **夜间范围**：`00:00–09:00`（与夜间相位一致：normal 段 = 09:00–24:00，即分钟 `540–1440`）。
- **推算规则**：从开始时间起，仅累计「白天 09:00–24:00」的分钟；落在夜间的时间段被跳过，跨到次日 09:00 继续。

**示例**（实现已验证）：
- 同盟大营（90 分钟）建于 `23:30`：当天剩余白天 `30` 分钟到 `24:00`，剩余 `60` 分钟于次日 `09:00` 起算 → 完成于次日 `10:00`。
- 攻城大营（60 分钟）建于 `23:30`：当天 `30` 分钟 + 次日 `09:00` 起 `30` 分钟 → 完成于次日 `09:30`。

> 注：甘特图视觉上仍把任务块按 `[startTime, endTime]` 连续绘制（即 23:30→次日 10:00 的整段色块），夜间暂停仅体现在「结束时间如何计算」，不影响色块渲染。

## 五、全部预设类型清单

> 每个模板的自动填充逻辑（`pickTemplate`）遵循统一规则：
> 1. 若 `durationMin > 0` → 大营按夜间暂停推算结束时间（`customEnd=false`），其余 `endTime = startTime + durationMin`；
> 2. 否则若 `pavingMode` 且 `countValue` → `endTime = startTime + round(countValue * PACE[pavingMode])`，`count = countValue`。
> （旧版「`countMode==='time'` 直接加分钟」分支已随「铺路（pave）」模板移除而废弃。）

### 1. 攻城大营（`siege`）
- **图标/颜色**：`fire-o` / `#ee0a24`
- **模板参数**：`durationMin:60`、`pavingMode:''`、`customEnd:false`
- **填充结果**：夜间暂停推算 → 开始时间 + 60 分钟有效白天时间（如 23:30 起 → 次日 09:30）
- **表单可见项**：名称、开始时间、结束时间（**仅展示**，不可改）、卡片颜色、优先级、备注、计数、高亮
- **规则**：仅填开始时间；结束时间由系统按夜间暂停推算并展示，用户不可编辑。

### 2. 同盟大营（`ally`）
- **图标/颜色**：`friends-o` / `#7232dd`
- **模板参数**：`durationMin:90`、`pavingMode:''`、`customEnd:false`
- **填充结果**：夜间暂停推算 → 开始时间 + 90 分钟有效白天时间（如 23:30 起 → 次日 10:00）
- **表单可见项**：同攻城大营（结束时间仅展示）
- **规则**：同攻城大营，仅固定时长改为 90 分钟。

### 3. 宣战（`declare`）
- **图标/颜色**：`warning-o` / `#ff976a`
- **模板参数**：`durationMin:60`、`customEnd:true`
- **填充结果**：`endTime = startTime + 60 分钟`（默认 1 小时，普通连续相加）
- **表单可见项**：名称、开始时间、结束时间（可改）、卡片颜色、优先级、备注、计数、高亮
- **规则**：默认 1 小时；结束时间可自由编辑（不再有最短时长 / 粒度吸附约束）。

### 4. 自动铺路（`auto-pave`）
- **图标/颜色**：`logistics` / `#07c160`
- **模板参数**：`pavingMode:'auto'`、`countMode:'count'`、`countValue:10`、`customEnd:true`
- **填充结果**：`PACE.auto = 3分10秒/格`（190 秒），`mins = round(10 * 3.1667) = 32 分钟` → `endTime = startTime + 32 分钟`，`count = 10`
- **表单可见项**：名称、开始/结束时间、铺路模式（默认自动铺路，可改）、填写方式切换、格数（`countMode=count` 时）、卡片颜色、优先级、备注、高亮
- **规则**：铺路模式锁定为自动铺路（仍可在弹层改）；默认填 10 格；改格数实时重算结束时间；改结束时间反推格数。

### 5. 接力铺路（`relay-pave`）
- **图标/颜色**：`exchange` / `#00b8d4`
- **模板参数**：`pavingMode:'relay'`、`countMode:'count'`、`countValue:10`、`customEnd:true`
- **填充结果**：`PACE.relay = 3分/格`（180 秒），`mins = round(10 * 3) = 30 分钟` → `endTime = startTime + 30 分钟`，`count = 10`
- **表单可见项**：同自动铺路（铺路模式默认接力铺路）
- **规则**：与自动铺路一致，仅节奏不同（3 分/格）。

### 6. 自定义事务（`custom`）
- **图标/颜色**：`add-square` / `#07c160`
- **模板参数**：`customEnd:true`（其余均空/默认）
- **填充结果**：无默认时长，进入纯手动编辑
- **表单可见项**：名称、开始/结束时间、铺路模式（默认「不铺路」，可改）、卡片颜色、优先级、备注、计数、高亮
- **规则**：`pavingMode` 为空；点击「铺路模式」可选择自动/接力铺路，选中后将按铺路逻辑补充「填写方式 / 格数」字段；`customEnd:true`。

## 六、铺路节奏常量（实现真值）

| 模式 | 每格基础分钟 | 白天/浅夜（08:00–次日 01:00）每格耗时 | 深夜（01:00–08:00）每格耗时 |
|------|------|------|------|
| `auto`（自动铺路） | 3 分 + 固定 10 秒行路 | **3 分 10 秒 / 格** | **9 分 10 秒 / 格（×3 + 固定 10s）** |
| `relay`（接力铺路） | 3 分（无附加） | **3 分 / 格** | **9 分 / 格（×3）** |

- 夜间（倍率）定义：`PAVING_NIGHT_START = 60`（01:00）、`PAVING_NIGHT_END = 480`（08:00）、`PAVING_NIGHT_MULT = 3`。
- **×3 只乘「3 分钟」基础部分**，自动铺路固定附加的 10 秒「行路」不随昼夜翻倍（即夜间 auto = 3×3 分 + 10 秒 = 9 分 10 秒）。
- 「当前格」的耗时由该格**起点**所处的昼夜决定（`paceSecAt(mode, absMin)`），逐格累加得到结束时间与累计格数。
- 核心函数集中在 `utils/taskTime.ts`（`pavedEndAbs` / `pavedCountBetween` / `paceSecAt`），`useTaskEditor.ts`、`utils/ganttViewBuild.ts`、`utils/ganttCanvasDraw.ts` 复用，三处口径一致。

## 七、实现差异说明（已同步）

原文档/注释与实际代码的不一致已在代码中统一，当前以代码为准：

1. **铺路节奏数值已对齐 9 分/格**
   - 早期代码曾用 `auto = 190 秒（3分10秒）`、`relay = 180 秒（3分）`，现已改为 `auto = 550 秒（9分10秒）`、`relay = 540 秒（9分）`，与 `README.md` / `types/gantt.ts` 注释一致。

2. **「夜间 ×3」已实现（仅铺路）**
   - 铺路在夜间（01:00–08:00）每格耗时 ×3，由 `paceSecAt` 逐格计算，影响：结束时间推算、卡片累计铺路格数、时间轴刻度数字。
   - 注意：**大营（攻城/同盟）的夜间行为是「暂停」而非「×3」**——大营在 `00:00–09:00` 不推进，结束时间按夜间暂停推算（见第四节），与铺路的 ×3 是两套独立机制。
   - 夜间相位（0–1 浅夜、1–8 深夜、8–9 浅夜）仍同时用于时间轴背景渐变与月亮图标展示。
